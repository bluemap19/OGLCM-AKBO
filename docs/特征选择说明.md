## 概述

OGLCM-AKBO 项目使用**定向灰度共生矩阵 (OGLCM)** 从测井图像中提取纹理特征，然后使用**贝叶斯优化自适配 K-means (AKBO)** 进行无监督聚类。

原始数据包含 **28 个纹理特征**，但通过前期分析发现，使用**4 个最优特征**即可达到良好的聚类效果，同时降低计算复杂度和噪声影响。

---

## 特征选择方法

### 随机森林特征重要性分析

我们使用**随机森林分类器**进行特征重要性分析：

1. **初步聚类：** 使用 K-means 对全部 28 个特征进行初步聚类 (K=5)
2. **伪标签生成：** 将聚类标签作为"伪标签"
3. **训练随机森林：** 训练随机森林分类器预测这些伪标签
4. **特征重要性排序：** 根据特征重要性得分排序
5. **选择 Top-4：** 选择最重要的 4 个特征

### 为什么选择随机森林？

| 优势 | 说明 |
|------|------|
| **非线性关系** | 能够捕捉特征与聚类之间的非线性关系 |
| **抗过拟合** | 集成学习方法，泛化能力强 |
| **特征交互** | 自动考虑特征之间的交互作用 |
| **稳定性** | 多次运行结果一致性好 |

## 最优特征列表

经过随机森林特征重要性分析，确定的**4 个最优特征**为：

| 排名 | 特征名 | 中文名称 | 地质意义 |
|------|--------|----------|----------|
| 1 | `CON_SUB_DYNA` | 对比度_子区域_动态 | 反映局部纹理变化强度，识别岩性边界 |
| 2 | `DIS_SUB_DYNA` | 差异性_子区域_动态 | 反映子区域灰度差异，区分不同沉积相 |
| 3 | `HOM_SUB_DYNA` | 同质性_子区域_动态 | 反映子区域纹理均匀度，识别均质储层 |
| 4 | `ENG_SUB_DYNA` | 熵_子区域_动态 | 反映纹理复杂度，表征层理发育程度 |

### 特征说明

**为什么这 4 个特征最重要？**

1. **都来自动态图像 (DYNA)** - 动态图像保留了更多原始地质信息
2. **都来自子区域 (SUB)** - 子区域分析捕捉了局部纹理特征
3. **覆盖不同纹理属性** - 对比度、差异性、同质性、熵，从多角度表征纹理

### 被剔除的特征

| 特征类型 | 原因 |
|----------|------|
| 静态图像特征 (STAT) | 信息量较少，与动态特征高度相关 |
| X/Y 方向特征 | 子区域特征已包含方向信息 |
| 其他纹理属性 | 重要性较低，可能引入噪声 |

## 代码适配说明

### 版本 2.1 变更 (2026-03-27)

**变更内容：**
- 移除了 `akbo_clustering.py` 中的随机森林特征选择功能

2. 减少计算开销，提高效率
3. 代码职责更清晰：数据预处理负责特征选择，聚类算法负责优化 K 值

## 使用示例

### 示例 1: 使用默认最优特征

```python
from src.data_loader import load_and_preprocess
from src.akbo_clustering import AKBOClusterer

# 加载数据 (自动选择 4 个最优特征)
depth, features, preprocessor, _ = load_and_preprocess('TZ1H_texture_logging.csv')

# 创建聚类器
clusterer = AKBOClusterer(
    k_range=(2, 10),
    n_init=5,
    max_iter=30,
    random_state=42
)

# 执行优化 (使用预设的 4 个最优特征)
selected_indices = [0, 1, 2, 3]  # 对应 CON_SUB_DYNA, DIS_SUB_DYNA, HOM_SUB_DYNA, ENG_SUB_DYNA
optimal_k = clusterer.optimize(
    features,
    feature_names=preprocessor.feature_columns,
    selected_indices=selected_indices
)

# 获取聚类标签
labels = clusterer.fit(features[:, selected_indices])
```

### 示例 2: 自定义特征选择

```python
# 如果想使用其他特征组合
custom_features = [
    'CON_SUB_DYNA',
    'HOM_SUB_DYNA',
    'ENG_SUB_DYNA'
]

# 获取特征索引

    preprocessor.feature_columns.index(feat)

]

# 使用自定义特征
optimal_k = clusterer.optimize(
    features,
    feature_names=preprocessor.feature_columns,
    selected_indices=selected_indices
)
```

## 常见问题

### Q1: 为什么不再在聚类器中进行特征选择？

**A:** 特征选择是数据预处理的一部分，应该在聚类之前完成。这样做的优势：
- 职责分离：数据预处理负责特征工程，聚类算法负责优化
- 效率更高：避免重复计算特征重要性
- 灵活性更好：可以根据不同数据集调整特征选择策略

### Q2: 如果想重新进行特征重要性分析怎么办？

**A:** 可以手动执行随机森林分析：

```python
from sklearn.ensemble import RandomForestClassifier
from sklearn.cluster import KMeans

# 初步聚类
kmeans = KMeans(n_clusters=5, random_state=42)
labels = kmeans.fit_predict(X)

# 训练随机森林
rf = RandomForestClassifier(n_estimators=100, random_state=42)
rf.fit(X, labels)

# 获取特征重要性
importances = rf.feature_importances_
ranking = np.argsort(importances)[::-1]

# 打印前 10 个重要特征
for i in range(10):
    print(f"{i+1}. {feature_names[ranking[i]]}: {importances[ranking[i]]:.4f}")
```

### Q3: 4 个特征是否足够？会不会丢失信息？

**A:** 根据实验结果：
- **聚类质量：** 使用 4 个最优特征的 UIndex 与使用 28 个特征相当甚至更好
- **计算效率：** 速度提升约 5-7 倍
- **可解释性：** 更容易地质解释

原因：
- 28 个特征之间存在高度相关性
- 许多特征包含冗余信息
- 噪声特征会降低聚类质量

### Q4: 如何验证特征选择的效果？

**A:** 可以对比实验：

```python
# 实验 1: 使用 4 个最优特征
optimal_k_4 = clusterer.optimize(X[:, :4])
metrics_4 = clusterer.best_metrics

# 实验 2: 使用全部 28 个特征
optimal_k_28 = clusterer.optimize(X)
metrics_28 = clusterer.best_metrics

# 对比
print(f"4 特征 UIndex: {metrics_4['uindex']:.4f}")
print(f"28 特征 UIndex: {metrics_28['uindex']:.4f}")
```
